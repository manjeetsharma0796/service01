# Klink MPP Echo Mock — research artifact

**Status**: research / spike artifact. **NOT** production Klink code.

This is a minimal MPP echo merchant — a Hono server that returns HTTP 402 challenges, accepts USDC payment on Solana (mainnet **and** devnet), and serves a dummy JSON response on retry. Built to:

1. Validate that the `mppx` + `@solana/mpp` server SDK works end-to-end for a Solana-method service.
2. Be **registrable on mpp.dev's directory** (via PR) and **MPPScan** (one-click) — see [Registration](#registration) below.
3. Serve as a parser-validation fixture for Klink's `/v1/spend/sign-payment` once it speaks MPP v2 (per [docs/memos/2026-05-09-mpp-tempo-migration-finding.md](../../memos/2026-05-09-mpp-tempo-migration-finding.md)).

The companion memo is **[docs/memos/2026-05-11-mpp-mock-service-and-registration.md](../../memos/2026-05-11-mpp-mock-service-and-registration.md)** — read that first for the strategic context (deployment options, registration steps, recommendation).

## What it does

- `GET /health` — free, returns `{ ok: true }`
- `GET /echo` — paid ($0.01 USDC), returns `{ message: "Paid!", timestamp, network, requestId }`
- `GET /openapi.json` — auto-generated by `discovery()`, includes `x-payment-info.offers[]` so MPPScan crawlers index it correctly

The `/echo` endpoint advertises **two** payment methods in a single 402 response:

```http
HTTP/1.1 402 Payment Required
WWW-Authenticate: Payment id="...", method="solana", intent="charge", request="<base64>", ...  # mainnet
WWW-Authenticate: Payment id="...", method="solana", intent="charge", request="<base64>", ...  # devnet
```

Each `request` field decodes to a JSON body with `methodDetails.network` set to `"mainnet"` or `"devnet"` respectively. A client picks the one matching its wallet's network.

## Requirements

- **Node 22+** (or Bun 1.2+)
- A Solana wallet with a published recipient pubkey (the address you want USDC sent to)
- (Optional) A 32-byte `MPP_SECRET_KEY` — used by mppx to sign challenge IDs; auto-generated if not set

## Setup

```bash
cd docs/research-artifacts/mpp-mock-service
cp .env.example .env
# Edit .env — set SOLANA_RECIPIENT to your wallet pubkey
bun install
bun run dev    # http://localhost:3000
```

Smoke-test:

```bash
# Health: should be 200 with {"ok":true}
curl -s http://localhost:3000/health

# Echo without payment: should be 402 with WWW-Authenticate headers
curl -is http://localhost:3000/echo

# Discovery doc: should include x-payment-info.offers[] for /echo
curl -s http://localhost:3000/openapi.json | jq '.paths["/echo"].get["x-payment-info"]'
```

## Deployment

### Option A — Render (recommended for a public, registrable URL)

```bash
# From repo root:
render deploy --service docs/research-artifacts/mpp-mock-service
```

Or use `render.yaml` in this folder (Render auto-detects on git push to a connected repo). Free tier is fine.

### Option B — Local + ngrok (fastest iteration)

```bash
bun run dev
ngrok http 3000
# Use the https://*.ngrok.app URL as your service base
```

### Option C — Cloudflare Workers (zero cold-start)

mppx supports Workers — see the [mpp.dev Workers guide](https://mpp.dev/quickstart). Adapter swap required (Hono → itty-router or Workers fetch).

## Registration

Once the service has a public HTTPS URL (e.g. `https://klink-echo.onrender.com`):

### 1. MPPScan — one-click

1. Open [mppscan.com/register](https://www.mppscan.com/register)
2. Paste the public URL (form has a single input)
3. Submit — MPPScan crawls your `/openapi.json` and indexes the discovery doc

### 2. mpp.dev/services — PR-based

1. Fork [github.com/tempoxyz/mpp](https://github.com/tempoxyz/mpp)
2. Add an entry to `services/llms.txt` (or wherever the directory file lives — confirm in the repo's CONTRIBUTING)
3. Open a PR

See the memo for screenshots and the exact PR template.

## What's missing for production

- **Recipient pubkey is a placeholder** in `.env.example`. Real deployment needs a wallet you actually control.
- **No on-chain settlement verification** — `mppx` does this by default for Solana, but with `network: "devnet"` you're inherently in test mode; mainnet flips the real-money switch.
- **No rate limiting** — fine for a mock, dangerous for anything else.
- **No HTTPS termination** — Render/Cloudflare handles this; local dev needs ngrok or Caddy.
